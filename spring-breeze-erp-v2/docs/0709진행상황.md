- 2차 추가 기능 부분
for_version 이용하여 양식 이름 또는 양식의 내용이 변경되었을때
버전을 올려 아예 새로 insert 하며
기존의 양식을 사용했던 데이터를 보존함

for_deleted 양식 을 삭제할시 버전과 같은 이유로
기존에 작성되어있는 문서들의 데이터에 영향을 끼칠수 있으므로
for_deleted = true로 설정하여 데이터는 남기고 사용할수 없게 만듬

2. 결재 문서 / appr_doc 파트
문서 작성, 조회, 승인/반려 까지 처리 되고 테이블에 데이터 잘 들어가는것도 확인
테스트할때 났던 오류들 수정까지 완료

- 2차 추가 기능 부분
doc_revision 결재 문서 기안 진행중에
현재는 기안 수정이 불가능 하지만 이후 기능 추가때
기안 수정중 승인 또는 반려가 먼저 진행된 경우 혹은 이 반대의 경우
처음 쿼리문이 작동 되었을때 revision의 수치가 1이 올라가며
나중에 작동한 쿼리문에서 revision의 수치를 비교하여 상태가 변경되었을때
동시성 제어로 데이터 꼬임을 방지함 / 임시로 컨트롤러 파트에서 강제로 revision수치를 다르게 잡아 작동확인함

3. 결재선 / appr_line 파트
문서 작성시에 들어가는
일반 결재시 1명, 중요문서 3명 까지 구현되고
순차적으로 결재하는것도 구현 완료
